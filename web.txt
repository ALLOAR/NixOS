Розделение сетей на подсети 

Давайте разберемся, как решать эти задачи по разделению на подсети. Для каждой из них нужно правильно подобрать маску подсети, которая позволит выделить требуемое количество сетей и хостов.

---

### 1. **194.12.13.0 /24 на 8 сетей**

Задача: Разделить сеть **194.12.13.0/24** на **8 сетей**.

- **Маска /24** — это стандартная маска подсети для сети с 256 адресами 
(от 194.12.13.0 до 194.12.13.255).
- Нам нужно разделить эту сеть на 8 подсетей. Для этого нам нужно взять дополнительные биты из 
адреса сети, чтобы создать нужное количество подсетей.

#### Как это делаем:

1. Число подсетей — **8**, значит нам нужно найти минимальное количество бит для подсетей. 
   - Чтобы получить хотя бы 8 подсетей, нужно взять 3 бита (2^3 = 8 подсетей).
   
2. Базовая маска — **/24**, добавляем 3 бита для подсетей, получаем маску **/27** (24 + 3 = 27).
обьяснение 3 бита взялись от того что 2^3 степени это 8, то есть те самые 8 подсетей которые нужны

3. Маска подсети: **255.255.255.224** (или **/27**).
Обьяснение 24 взялось от последнего октета, если записать маску 27 в двоичном виде то получиться 

255.255.255.11100000  (/27) и последний окетет как раз и есть 224 в десятичном виде
4. Сеть будет выглядеть так:

- 194.12.13.0/27 (первая подсеть)
- 194.12.13.32/27 (вторая подсеть)
- 194.12.13.64/27 (третья подсеть)
- 194.12.13.96/27 (четвертая подсеть)
- 194.12.13.128/27 (пятая подсеть)
- 194.12.13.160/27 (шестая подсеть)
- 194.12.13.192/27 (седьмая подсеть)
- 194.12.13.224/27 (восьмая подсеть)

Каждая подсеть будет иметь **30 доступных хостов** (2^5 - 2 = 30), 
так как два адреса в подсети зарезервированы (адрес сети и адрес широковещательной рассылки).

### 2. **172.16.0.0 /21 по 255 хостов**

Задача: Разделить сеть **172.16.0.0/21** так, чтобы в каждой подсети было **255 хостов**.

- **Маска /21** — это сеть с 2048 адресами (от 172.16.0.0 до 172.16.7.255).
- Нам нужно подсчитать, сколько бит нужно для того, чтобы было по 255 хостов в каждой подсети.

#### Как это делаем:

1. Чтобы получить **255 хостов**, нам нужно 8 бит для хостовой части (2^8 - 2 = 254 хоста). 
Это минимальное количество для того, чтобы поместить 255 хостов.
   
2. Маска сети: **/24**. То есть, нам нужно взять **3 бита** из хостовой части (так как 24 + 3 = 27). 

3. Получаем маску **/24**, что соответствует маске **255.255.255.0**.

4. Сеть будет выглядеть так:

- 172.16.0.0/24
- 172.16.1.0/24
- 172.16.2.0/24
- 172.16.3.0/24

Каждая подсеть будет содержать 254 хоста.

### 4. **200.3.1.0 /25 по 15 хостов**

Задача: Разделить сеть **200.3.1.0/25** так, чтобы в каждой подсети было **15 хостов**.

- **Маска /25** — это сеть с 128 адресами (от 200.3.1.0 до 200.3.1.127).
- Нам нужно подсчитать, сколько бит нужно для того, чтобы было по 15 хостов в каждой подсети.

#### Как это делаем:

1. Чтобы получить **15 хостов**, нам нужно 4 бита для хостовой части (2^4 - 2 = 14 хостов). Это минимальное количество для того, чтобы поместить 15 хостов.
   
2. Маска сети: **/28**. То есть, нам нужно взять **3 бита** из хостовой части (так как 25 + 3 = 28).

3. Получаем маску **/28**, что соответствует маске **255.255.255.240**.

4. Сеть будет выглядеть так:

- 200.3.1.0/28
- 200.3.1.16/28
- 200.3.1.32/28
- 200.3.1.48/28
- 200.3.1.64/28
- 200.3.1.80/28
- 200.3.1.96/28
- 200.3.1.112/28

Каждая подсеть будет содержать **14 хостов**.
**Резюме**:
- Каждую сеть необходимо разбить на требуемое количество подсетей, корректно изменив маску.
- Важно учитывать количество бит, которые нужны для хостовой и сетевой части.
- После разделения сети нужно правильно распределить подсети по диапазонам адресов.

32 не является размером маски, а представляет собой общее число бит в IPv4-адресе. При работе с IPv4 мы всегда имеем 32 бита. Размер подсети (то есть количество адресов в ней) рассчитывается по формуле:

  Количество адресов = 2^(32 – N),

где N — длина маски (префикс). Например, для маски /27 получаем 2^(32–27) = 2^5 = 32 адреса, а для маски /26 – 2^(32–26) = 2^6 = 64 адреса. В IPv6 общее число бит равно 128.

Таким образом, число 32 постоянно для IPv4, но фактический размер блока адресов зависит от того, сколько бит вы оставляете для адресации хостов.

Количество хостов в подсети вычисляется по формуле:

  **2^(32 – N) – 2**

где **N** – длина маски (число бит, отведённых под сеть).

Например, для маски **/27**:
- Остаток бит для хостовой части: 32 – 27 = 5
- Всего адресов: 2^5 = 32
- Доступных для хостов (после вычитания адреса сети и широковещательного адреса): 32 – 2 = 30

Для маски **/26**:
- Остаток бит: 32 – 26 = 6
- Всего адресов: 2^6 = 64
- Доступных для хостов: 64 – 2 = 62
